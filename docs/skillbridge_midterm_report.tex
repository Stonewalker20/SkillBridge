\section{Project Description}

\subsection{Project Completion Level}
SkillBridge is approximately \textbf{55\%--60\% complete} at midterm. The project currently has \textbf{12 implemented use cases} with real data and active database persistence. These use cases cover the core value of the product: collecting evidence, extracting and confirming skills, analyzing job fit, identifying skill gaps, generating tailored resumes, and providing user analytics. Login, logout, sign in, and sign up are implemented but are not counted toward the required feature total.

\subsection{System Overview}
SkillBridge is a full-stack career intelligence platform that helps users turn resumes, projects, coursework, and work artifacts into a structured, evidence-backed skill profile. The system then compares that profile against target job postings, identifies skill gaps, recommends learning paths, and generates tailored resumes for specific opportunities.

\subsection{Targeted Audience}
The primary target audience is students, early-career professionals, career changers, and job seekers who want to understand how their skills map to real jobs. A secondary audience is administrators or product owners who need visibility into user activity, moderation workflows, and platform health.

\subsection{Similar Product}
Products similar to SkillBridge include LinkedIn, Teal, Resume Worded, Handshake, and career readiness platforms that help users optimize resumes and compare their experience to job postings. SkillBridge differs by combining evidence-backed skill confirmation, job-match analysis, local AI tooling, skill trajectory modeling, and tailored resume generation in one product.

\subsection{Source of Revenue}
Potential revenue sources for SkillBridge include:
\begin{itemize}[leftmargin=1.25em]
  \item subscription plans for premium analytics and tailored resume generation
  \item university or institutional licensing
  \item employer-facing recruiting and candidate matching dashboards
  \item premium job intelligence and learning-path recommendations
\end{itemize}

\subsection{User Types}
SkillBridge currently includes at least two user types:
\begin{itemize}[leftmargin=1.25em]
  \item \textbf{Standard User:} manages evidence, skills, job matches, analytics, resumes, and account settings
  \item \textbf{Owner/Admin/Team User:} has access to all standard-user features plus the Admin Workspace for moderation, role management, and system oversight
\end{itemize}

\section{Software Architecture}

\subsection{Subsystems and Features}
The project currently includes more than the required five subsystems:
\begin{enumerate}[leftmargin=1.5em]
  \item \textbf{Authentication and Account Management}
  \begin{itemize}[leftmargin=1.25em]
    \item user registration and login
    \item session-based authentication
    \item profile avatar upload and preset selection
    \item account preferences and theme customization
  \end{itemize}
  \item \textbf{Skills Management}
  \begin{itemize}[leftmargin=1.25em]
    \item browse skill catalog
    \item add custom skills
    \item confirm extracted skills
    \item update skill proficiency
  \end{itemize}
  \item \textbf{Evidence Management}
  \begin{itemize}[leftmargin=1.25em]
    \item paste evidence text
    \item upload PDF and DOCX evidence
    \item multi-file analysis
    \item save and edit evidence
  \end{itemize}
  \item \textbf{Dashboard and Analytics}
  \begin{itemize}[leftmargin=1.25em]
    \item dashboard summary metrics
    \item recent activity feed
    \item top skill categories
    \item skill trajectory and learning path analytics
  \end{itemize}
  \item \textbf{Job Match Analysis}
  \begin{itemize}[leftmargin=1.25em]
    \item ingest job posting
    \item compute matched and missing skills
    \item semantic alignment and keyword overlap
    \item gap reasoning and reanalysis
  \end{itemize}
  \item \textbf{Tailored Resume Generation}
  \begin{itemize}[leftmargin=1.25em]
    \item preview tailored resume
    \item generate PDF output
    \item save tailored resumes
    \item reopen prior resume generations
  \end{itemize}
  \item \textbf{Admin Workspace}
  \begin{itemize}[leftmargin=1.25em]
    \item system overview metrics
    \item user role management
    \item job moderation workflows
  \end{itemize}
\end{enumerate}

\subsection{Technology}
SkillBridge uses the following technology stack:
\begin{itemize}[leftmargin=1.25em]
  \item \textbf{IDE / Development Environment:} Visual Studio Code and a local terminal-based workflow
  \item \textbf{Backend:} FastAPI with Python 3.12
  \item \textbf{Frontend:} React with Vite and TypeScript
  \item \textbf{Database:} MongoDB
  \item \textbf{Version Control:} Git and GitHub
  \item \textbf{Testing:} Pytest for backend route and contract testing
  \item \textbf{Local AI Components:} local transformer embeddings, retrieval-augmented grounding, local LLM-backed rewrite fallback pipeline
\end{itemize}

\section{Appendix A: System Vision Document}
\subsection{Problem Statement}
Job seekers often have fragmented evidence of their skills and limited visibility into how their background aligns with specific jobs.

\subsection{Vision}
SkillBridge will help users turn evidence into structured skills, compare their profile to jobs, identify career gaps, and generate targeted resumes using local intelligence.

\subsection{Objectives}
\begin{itemize}[leftmargin=1.25em]
  \item centralize user evidence and skill confirmation
  \item provide explainable job-match analysis
  \item support learning paths and career trajectory planning
  \item generate tailored resumes from real user evidence
\end{itemize}
